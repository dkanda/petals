# Project Structure

## Root Directory
- `agents.md`: Documentation regarding agents.
- `src/`: Source code for the project.
- `test/`: Unit tests and integration tests.
- `docs/`: Additional documentation and resources.

## Source Code
The `src/` directory contains the following components:
- `main.py`: The entry point of the application.
- `models.py`: Data models used in the application.
- `services/`: Business logic and service classes.

## Tests
The `test/` directory includes:
- `test_main.py`: Tests for the main application logic.
- `test_services.py`: Tests for service classes.

# Coding Standards
- Follow [PEP 8](https://www.python.org/dev/peps/pep-0008/) for Python code style.
- Use descriptive variable and function names.
- Include docstrings for all modules, classes, and methods.
- Limit line length to 79 characters.

# Development Guidelines
- Branch naming convention: `feature/your-feature-name` for new features, `bugfix/your-bugfix-name` for bug fixes.
- Ensure all tests pass before making a pull request.
- Write clear commit messages that reflect the changes made.
- Review the code of others before merging to maintain code quality and consistency.

## Contribution
To contribute to this project, please follow these steps:
- Fork the repository.
- Create a new branch for your feature or bugfix.
- Make your changes and commit them with informative messages.
- Submit a pull request with a description of your changes.
